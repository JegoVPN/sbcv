<!-- reviewer: senior PM + principal FE; single source of truth: sing-box testing 1.14 -->

Upstream authority: `docs/upstream/sing-box/testing/configuration/dns/server/tailscale.md`. That doc defines exactly 5 writable keys: `type` (fixed `"tailscale"`), `tag`, `endpoint` (Required), `accept_default_resolvers` (bool, default `false`), `accept_search_domain` (bool, default `false`, since 1.14.0). No `server`/`server_port`/`path`/`address`/`detour`/`tls` exist on this type — it is endpoint-backed MagicDNS.

## Verdict (2-3 sentences)
The node is wired correctly end-to-end for the 1.12 surface: palette entry exists, the canvas links to a Tailscale endpoint, a hard diagnostic blocks a missing/dangling `endpoint`, and the Inspector exposes `endpoint` + `accept_default_resolvers` as first-class controls. The two remaining real defects are: (P0) `accept_search_domain` (the 1.14.0 field this whole doc revision is about) has NO first-class Inspector control and is never seeded by `createDnsServer`, so on the testing channel it is silently unreachable; and (P1) the always-on "Detour outbound" right port on the tailscale node lets a user write a non-canonical `detour` key. Pass-1's biggest claims (dangling `endpoint: "tailscale-ep"` default, `accept_default_resolvers` buried in Advanced, no missing-endpoint diagnostic) are now STALE — all three are fixed in code.

## Findings (prioritized)
- **[P0-1] `accept_search_domain` has no first-class control and is never seeded.** `Inspector.tsx:4352-4378` (tailscale sub-block omits it) + `commands.ts:669-675` (`createDnsServer` omits it) + `Inspector.tsx:241-260` (`dnsServerHandledFields` omits it). On the testing channel this is the single new field of the 1.14 doc revision and a user cannot enable it from the UI. Fix: render a testing-channel-gated toggle next to `accept_default_resolvers` (the `channel` value is already in scope at `Inspector.tsx:1750`), e.g. `{entityType === "tailscale" && channel === "testing" ? <toggle accept_search_domain> : null}`; optionally seed `accept_search_domain: false` behind the testing channel.
- **[P1-1] Spurious "Detour outbound" right port on the tailscale node writes a non-canonical `detour` key.** `portRelationRegistry.ts:105` (`dns-server-detour` source endpoint has no `nodeType`, so it matches tailscale) → port rendered `SbcNode.tsx:346-381`, edge `graph.ts:551-553`, write `useProjectStore.ts:574-575`. Tailscale has no `detour` field. Fix: add `nodeTypeExcludes: ["tailscale","resolved","hosts","fakeip","dhcp"]` (or a positive include set) to the `dns-server-detour` source endpoint so the port disappears for endpoint-/path-backed server types.
- **[P1-2] No inline "create endpoint" affordance when zero tailscale endpoints exist.** `Inspector.tsx:4360` shows placeholder "Create or select endpoint" but the option is inert; with no `endpoint:tailscale` in the config the select is empty and the only recovery is leaving the Inspector. Fix: add a small "Create Tailscale endpoint" button beside the select that calls the existing add-endpoint path and wires `endpoint` (mirrors `useProjectStore.ts:760-762`).
- **[P2-1] `endpoint` lacks a required-field marker in the Inspector.** `Inspector.tsx:4354` label "Tailscale Endpoint" has no `*`/required styling even though it is the only Required field on the type and is hard-blocked by diagnostics. Fix: add a required indicator and surface the existing `dns-server-tailscale-endpoint-missing` diagnostic inline on the field.
- **[P2-2] `accept_default_resolvers` helper copy is slightly inaccurate.** `Inspector.tsx:4376` says "forward queries to MagicDNS chain"; upstream says it controls accepting default resolvers for *fallback* queries *in addition to* MagicDNS, and that when off, non-Tailscale domains return `NXDOMAIN`. Fix: align copy with the doc.

Stale pass-1 items (now fixed in code, do not re-file): P0-1 "buried in Advanced" — now a first-class toggle (`Inspector.tsx:4368`). P0-2 dangling `endpoint: "tailscale-ep"` default — `createDnsServer` no longer sets `endpoint` (`commands.ts:669-675`) and a missing-endpoint error diagnostic exists (`diagnostics.ts:1116-1123`). P0-3 detour port — still valid, re-filed here as P1-1. P0-4 palette gating / auto-wire — reverse + forward wiring now present (`useProjectStore.ts:760-762`,`:788-790`,`:824-825`).

SUMMARY: 1 P0, 2 P1, 2 P2.
