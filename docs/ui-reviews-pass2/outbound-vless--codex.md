## Verdict
The VLESS outbound is usable for the common happy path: Palette adds a real canonical `type: "vless"` outbound, the canvas exposes the main outbound reference ports, and the Inspector now has first-class controls for server, port, UUID, flow, network, packet encoding, TLS, multiplex, transport, and dial fields. It is not yet faithful to the testing 1.14 model end-to-end: the shared TLS card exposes server-only fields that can write invalid outbound JSON, misses several testing 1.14 outbound TLS fields, and the V2Ray transport card is not type-aware. Stable and testing VLESS top-level docs are identical; the version-specific gaps below come from shared TLS/Dial/Rule Set docs.

## Findings (prioritized)

Split inbound/outbound TLS definitions instead of using one mixed card — [src/components/Inspector.tsx:1521](/Users/ejan/SGP/sbc-ui/.claude/worktrees/docs+claude-pre-push-review-gate-spec/src/components/Inspector.tsx:1521), [src/components/Inspector.tsx:1543](/Users/ejan/SGP/sbc-ui/.claude/worktrees/docs+claude-pre-push-review-gate-spec/src/components/Inspector.tsx:1543).
- [P1] V2Ray transport UI is not type-aware and misses official fields. It always renders `host` as a list and always shows generic fields, which can write the wrong type for `httpupgrade.host`; it also omits `method`, `headers`, `max_early_data`, `early_data_header_name`, and `permit_without_stream`. Render fields by `transport.type` with correct string/list/object controls — [src/components/Inspector.tsx:1578](/Users/ejan/SGP/sbc-ui/.claude/worktrees/docs+claude-pre-push-review-gate-spec/src/components/Inspector.tsx:1578).

Add outbound TLS-only controls with channel/version copy where needed — [src/components/Inspector.tsx:1502](/Users/ejan/SGP/sbc-ui/.claude/worktrees/docs+claude-pre-push-review-gate-spec/src/components/Inspector.tsx:1502).
