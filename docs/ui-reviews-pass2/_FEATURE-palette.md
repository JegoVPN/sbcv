# Feature UX review — Palette / Add Library
<!-- reviewer: principal PM + FE; lens: can a new user understand & use it; source: our code + UX principles -->

## Feature inventory
- Panel titled "Add Library", fixed left, scrollable, ~286px wide — `Palette.tsx:326-327`, `styles.css:568-583`.
- Search box "Search config" filtering entries by group title + label — `Palette.tsx:328-342`, filter logic `Palette.tsx:310-321`.
- Two collapsible "mode pills": **Templates** (count of presets) and **Library** (count of groups); mutually exclusive, both start collapsed — `Palette.tsx:343-370`, state defaults `Palette.tsx:291-293`.
- Templates section: 9 one-click full-config presets; loading one replaces the whole project and marks it "Added" — `Palette.tsx:62-72`, `templates.ts:1330-1403`, store `useProjectStore.ts:684-697`.
- Library nav: 14 group buttons (Log, DNS, NTP, Certificate, Cert Providers, HTTP Clients, Endpoints, Inbounds, Outbounds, Route, Services, Experimental, Shared) each expanding one section — `Palette.tsx:385-400`.
- ~120 entry buttons, each = icon + label + status badge + a separate "Docs" link (library only) — `Palette.tsx:458-487`.
- 9 status types driving badge text, color, tooltip, and enabled/disabled — `Palette.tsx:43`, `statusLabel:240-250`, `itemStatus:258-265`, `canActivate:279-287`.
- Click action: templates → `loadTemplatePreset`; everything else → `createFromPalette(kind)`; singletons also select their node — `Palette.tsx:472-476`, store `createFromPalette:708-796`.
- Dynamic status: deprecated set (`block`, `hysteria-out`, `dns-fakeip`), singleton "open" when already in config, channel-gated `service-hysteria-realm` — `Palette.tsx:252-265`, `singletonsPresent:300-309`.

## UX findings (prioritized)
- **[P0] Empty first-run: panel shows nothing actionable.** Both pills default collapsed (`Palette.tsx:291-293`), so a newcomer sees only a search box + "Templates"/"Library" pills and zero nodes. The big "Add Library" title implies content that isn't there; the user must guess to click a pill. A canvas editor's add-panel should reveal at least one populated group (or Templates) by default.
- **[P0] ~10 entries look addable but are dead controls.** Items with neither `status` nor `ready` silently resolve to `docs` and render disabled (45% opacity only): `dns-fakeip`, `dns-legacy`, `dns-mdns`, `wireguard-out`, `dns-out`, `route-geoip`, `route-geosite`, `rule-set-source-format`, `rule-set-adguard`, `route-sniff` (`Palette.tsx:82-188`, fallthrough `itemStatus:264`). They sit in the same list as working rows; the only signal is a faint gray "Docs" badge + tooltip. A user clicks WireGuard/DNS outbound and nothing happens.
- **[P0] Permanently-gated entries can never be added here, with no path forward.** All Certificate Providers, HTTP Client, and many Shared rows are `status:"gated"` (`Palette.tsx:110-118,218-232`); they're disabled and the tooltip says "target-gated and needs matching sing-box validation" (`statusTitle:272`) — opaque to a newcomer and offers no next step. The panel advertises features the user literally cannot use from it.
- **[P1] Nine status badges are jargon and several are unguessable.** "Setup", "Table", "Inspector", "Docs", "Gated", "Pending", "Legacy", "Open" (`statusLabel:240-250`) are config-tool internals. A first-timer cannot tell that "Setup" adds a node, "Inspector" means edit-it-elsewhere, "Table"/"Open" sometimes click and sometimes don't (`canActivate:279-287`). Badge color is the only differentiator and is unlabeled.
- **[P1] "Add" vs "Setup" distinction is invisible but both add a node.** Both are actionable and both drop a node on the canvas (`canActivate:280-281`); the difference (ready vs draft-with-placeholders) lives only in a hover tooltip (`statusTitle:268-269`). Users won't know why "Mixed" says Add but "SOCKS" says Setup when both create an inbound.
- **[P1] Search ignores Templates entirely.** `filteredGroups` maps over `libraryGroups` only (`Palette.tsx:313-321`); typing "1.14", "official", or "bypass" returns nothing even though presets match (`templates.ts:1330-1403`). Newcomers most likely to search are the ones who'd benefit most from finding a template.
- **[P1] Templates have no Docs link or description.** The Docs link is suppressed for the template group (`Palette.tsx:482`), yet labels like "1.12 community template" / "official client bypass no leak" are the most opaque in the whole panel and replace the entire project on click — a destructive, unexplained action with no "what is this?" affordance.
- **[P1] No add affordance is communicated; no drag despite a node canvas.** Entries are click-only (no `draggable`/`onDragStart` anywhere in `Palette.tsx`), but in a React-Flow editor users expect to drag onto the canvas. Nothing states "click to add"; after a click the node appears off-screen at pinned coordinates (e.g. `-300,40` for settings, `useProjectStore.ts:715`) with no scroll-to or toast, so feedback is easy to miss.
- **[P2] Icon inconsistency across the protocol set.** The same protocol gets different/generic icons by section and Globe2/Server/Shield are heavily overloaded — e.g. HTTP, DNS, HTTPS, Naive, Cloudflared all use `Globe2`; Tor + SOCKS share `Network`; nearly every DNS server is `Server`/`Globe2` (`Palette.tsx:80-233`). Icons add little discriminating value, matching the icon-inconsistency class of bug found elsewhere.
- **[P2] 14 groups / ~120 entries with no "start here" tier.** Deep internals (Headless Rule, Pre-match, DNS01 Challenge, Listen/Dial Fields, Neighbor Resolution — `Palette.tsx:213-233`) sit at the same level as core building blocks. No "common nodes" grouping or onboarding ordering; the firehose buries the 5–6 things a beginner actually needs.
- **[P2] Group/label casing and naming is inconsistent.** Library labels are Title Case while template labels are lowercase ("official client tun ipv4", `templates.ts:1357`); parenthetical gates like "(with_tailscale)", "(Linux only)", "(1.14 testing)" leak raw flag names into labels (`Palette.tsx:95,124,145,200`).

## New-user verdict
A newcomer opening "Add Library" first sees an empty panel (both sections collapsed) and has to discover the pills before anything appears — a rough start. Once expanded, the panel is a dense, jargon-badged list where roughly a third of rows are disabled or gated with no clear reason, and the most newcomer-friendly content (Templates) is hidden from search and stripped of any explanation. Biggest friction: there is no visible difference between "I can add this", "I edit this elsewhere", and "this does nothing here" beyond tiny colored uppercase words and hover tooltips, so first-timers will click dead rows and misread the status vocabulary.

SUMMARY: 3 P0, 5 P1, 3 P2.
