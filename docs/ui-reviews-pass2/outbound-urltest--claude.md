A typo like `3min` round-trips silently and sing-box rejects it at runtime. Consider a lightweight duration regex hint/diagnostic. (`tolerance` already validates numerically; `url` already has scheme diagnostics.)
- [P2] Dead legacy comma-separated `outbounds` text fallback still present in JSX (`Inspector.tsx:4126-4133`). Unreachable for urltest while `tagValue!==null` (the checklist branch wins), but it writes raw `fromList` strings with no self-exclusion and is reachable for an untagged group. - [P2] Big `+` add on a urltest node always creates a `direct` candidate (first compatible chip, `graph.ts:408`, `useProjectStore.ts:802`); the empty-state `outbound-member` port toggle hard-codes `socks` (`useProjectStore.ts` group-member branch). For a latency-test group the most useful default is an existing proxy, not a fresh stub. Per-chip add and the Inspector checklist both cover real cases, so cosmetic. (Pass-1 P1-B's "SOCKS-only" claim is STALE; the chip set is now complete.)
- [P2] Type-switch silently discards `outbounds`/`url`/`interval`/`tolerance`/`idle_timeout`/`interrupt_exist_connections` when leaving/entering urltest (`commands.ts:913-919` keeps only `detour`). No confirm or diagnostic. Low priority but lossy.
